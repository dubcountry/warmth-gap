# The Warmth Gap

An interactive data tool on winter indoor comfort and energy hardship across the EU, US and Canada. The thesis is that latitude does not decide who stays warm. Income, building stock and fuel do. The centerpiece "Warm for whom?" view contrasts indoor temperature in smart-thermostat homes (tado) against the share of the whole population unable to keep warm (Eurostat), exposing the gap between the comfort of the connected few and the cold of everyone else.

Built as a static site, Economist data-journalism style, with Plotly. The Eurostat layer is data-driven from a small Python pull; the tado layer is the published appendix, embedded.

## What is in here

```
warmth-gap/
  public/                     <- the deployable static site (this is the web root)
    index.html                <- the whole app: markup, styles, logic, embedded tado data
    data/
      thermal_data.json       <- generated by the pull (not committed until you run it)
  scripts/
    eurostat_pull.py          <- Eurostat ETL -> public/data/thermal_data.json
  .github/workflows/
    refresh-data.yml          <- optional: regenerate the data yearly and commit it
  package.json                <- dev server, data, and deploy scripts
  .gitignore
  CLAUDE.md                   <- context primer for Claude Code
  README.md
```

The app renders immediately with built-in sample values. Once you run the pull, it loads the full Eurostat series for every reporting country and the year slider turns on.

## Run it locally

Requires Python 3 and (for the deploy script) Node. No Python packages needed.

```bash
cd warmth-gap

# 1. generate the Eurostat data (needs internet; writes public/data/thermal_data.json)
python3 scripts/eurostat_pull.py
#   sanity check the decoder without any network:
#   python3 scripts/eurostat_pull.py --selftest

# 2. serve the site (a server is required; opening the file directly blocks the data fetch)
python3 -m http.server 8000 --directory public
#   then open http://localhost:8000
```

With npm:

```bash
npm run data    # python3 scripts/eurostat_pull.py
npm run dev     # serve public/ on http://localhost:8000
```

## Put it online

The site is fully static, so any static host works. Two routes for Cloudflare.

### Route A: GitHub plus Cloudflare Pages (auto-deploy on push)

```bash
cd warmth-gap
git init
git add .
git commit -m "warmth gap: initial site"
git branch -M main
git remote add origin git@github.com:emonabiogit/warmth-gap.git   # create the empty repo first
git push -u origin main
```

Then in the Cloudflare dashboard, Workers and Pages, Create, Pages, Connect to Git, pick the repo, and set:

- Framework preset: None
- Build command: leave empty
- Build output directory: `public`

Save and deploy. Every push to `main` redeploys. Important: commit `public/data/thermal_data.json` so the deployed site has real data, since Cloudflare will not run the Python pull by default. If you prefer Cloudflare to refresh the data itself, use the GitHub Action in `.github/workflows/refresh-data.yml`, which regenerates and commits the file on a schedule.

### Route B: Wrangler (deploy straight from your machine)

```bash
npx wrangler pages deploy public --project-name=warmth-gap
# or: npm run deploy
```

First run will prompt you to log in to Cloudflare and create the project.

## Data and caveats

- Eurostat `ilc_mdes01` (unable to keep warm), `nrg_chdd_a` (heating degree days), `nama_10_pc` (GDP per capita), pulled live via the dissemination API.
- tado "The Cost of Comfort", February 2026, full country appendix. These are smart-thermostat homes, an affluent subset that is not representative of the population. That bias is the point of the divergence view, not a flaw to hide.
- The EU, US and Canada hardship measures use different definitions and are not directly comparable. The "Three countries" view is built to show why.
